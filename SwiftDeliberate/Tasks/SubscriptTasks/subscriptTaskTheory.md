
🎒 ПРАКТИЧЕСКИЙ КЕЙС: УМНЫЙ РЮКЗАК ГЕРОЯ (SAFE SUBSCRIPT)

Реальная проблема в Swift: 
    Обычный массив в Swift («рюкзак») при запросе несуществующего индекса моментально роняет приложение (краш). Наша цель — написать сабскрипт-«вышибалу», который проверяет индекс до того, как трогать массив.Наш рюкзак в памяти (Массив): ["🗡️ Меч", "🛡️ Щит", "🧪 Зелье"]
    Всего предметов в рюкзаке: 3Разрешенные индексы: 0, 1, 2📝 
    
1. ПОНЯТНЫЙ ПСЕВДОКОДБлок GET (Когда игрок забирает вещь из рюкзака, чтобы использовать)textЗапрос: Интерфейс игры дает нам Число-Индекс (например, индекс 10).

ЕСЛИ (Индекс больше или равен 0) И (Индекс меньше, чем 3):
    ➔ ПЛАН А (Индекс безопасен):
       Заглядываем в рюкзак, берем реальный предмет под этим номером.
       Отдаем предмет на экран (например, персонаж берет "🛡️ Щит").
ИНАЧЕ:
    ➔ ПЛАН Б (Фоллбек - Индекс опасен!):
       В рюкзак НЕ заглядываем (иначе игра вылетит!).
       Просто отдаем экрану текст "🔲 Empty Slot".
Используйте код с осторожностью.Блок SET (Когда игрок нашел новую вещь и хочет положить в рюкзак)textЗапрос: Нам дают Число-Индекс (куда положить) и Новую Вещь (newValue = "💍 Кольцо").

ЕСЛИ (Индекс больше или равен 0) И (Индекс меньше, чем 3):
    ➔ ПЛАН А (Индекс безопасен):
       Заходим в рюкзак в указанную ячейку.
       Выбрасываем то, что там лежало, и записываем туда "💍 Кольцо".
ИНАЧЕ:
    ➔ ПЛАН Б (Индекс опасен!):
       В рюкзак НЕ лезем.
       Ничего не делаем, команду записи просто игнорируем.

💻 3. ПЕРЕВОД НА СВИФТ (Код для Xcode)А теперь посмотрите, как красиво этот псевдокод и схемы превращаются в реальный код на Swift:swiftstruct HeroInventory {
    // Наш рюкзак с 3 предметами
    var items: [String] = ["🗡️ Меч", "🛡️ Щит", "🧪 Зелье"]
    
    subscript(index: Int) -> String {
        get {
            // Ромб проверки (Индекс >= 0 И Индекс < количества элементов)
            guard index >= 0 && index < items.count else {
                return "🔲 Empty Slot" // 🔴 План Б (Фоллбек)
            }
            
            return items[index] // 🟢 План А (Безопасно отдаем вещь)
        }
        set {
            // Такой же ромб проверки для записи
            guard index >= 0 && index < items.count else {
                return // 🔴 План Б: Индекс опасен, просто выходим и ничего не делаем
            }
            
            // 🟢 План А: Перезаписываем вещь (newValue — это то, что мы кладем)
            items[index] = newValue
        }
    }
}
